## 1. プロジェクト概要

### 1.1 プロジェクト名
**DX Consulting CRM - 案件管理システム**

### 1.2 背景
DXコンサルタントとして、以下の業務を日常的に実施しています※仮
- 顧客からの課題ヒアリング
- DX戦略書の策定
- セミナーの企画・実施
- カリキュラムの提案
- 個別相談への対応
- 新規顧客へのインサイドセールス
- エンジニアへの技術的な引継ぎ

これらの業務において、案件情報が散在し、進捗管理やチーム連携に課題があります！

### 1.3 プロジェクトの目的
DXコンサルタントの業務効率化とチーム連携の強化を目的とした、
案件管理システムの開発を行います。

### 1.4 対象ユーザー
- **主要ユーザー**: DXコンサルタント
- **副次ユーザー**: エンジニア（@Staff_Notsu氏、浦上泰弘氏等）

### 1.5 プロジェクト期間
- **開発期間**: 2025/11/21
- **その他**: ポートフォリオ用途として作成  
- **お試し**:Githubからクローンしていただき、初期設定を進めていただければ即時利用可能です。

## 2. 現状分析と課題

### 2.1 現状の業務フロー※一次面接前のイメージです！

```
顧客相談受付
    ↓
ヒアリング実施（メール・電話・対面）
    ↓
課題分析・整理（Excel・メモ・メール）
    ↓
提案書作成（Word・PowerPoint）
    ↓
見積提示（Excel）
    ↓
商談・調整（メール・電話）
    ↓
エンジニアへの引継ぎ（口頭・メール）
    ↓
プロジェクト実施
```

### 2.2 現状の課題※仮

#### 課題1: 情報の散在
- **現象**: 案件情報がメール、Excel、メモに分散
- **影響**: 情報を探すのに時間がかかる、過去案件の参照が困難
- **重要度**: ★★★★★

#### 課題2: 進捗管理の困難
- **現象**: 複数案件を並行管理する際、全体像が把握しづらい
- **影響**: 対応漏れ、優先度判断の遅れ
- **重要度**: ★★★★☆

#### 課題3: チーム連携の非効率
- **現象**: エンジニアへの引継ぎが口頭やメールで煩雑
- **影響**: 引継ぎ漏れ、技術要件の認識齟齬
- **重要度**: ★★★★★

#### 課題4: ナレッジの属人化
- **現象**: 過去の提案内容や対応履歴が個人に蓄積
- **影響**: チーム全体でのナレッジ共有ができない
- **重要度**: ★★★☆☆

#### 課題5: レポーティングの手間
- **現象**: 月次報告のために手動で集計作業
- **影響**: 報告資料作成に多大な時間を要する
- **重要度**: ★★★☆☆

### 2.3 課題の定量化の例

| 項目 | 現状 | 目標 | 改善率 |
|------|------|------|--------|
| 案件情報検索時間 | 平均10分/回 | 1分/回 | 90%削減 |
| 進捗確認時間 | 30分/日 | 5分/日 | 83%削減 |
| 引継ぎ準備時間 | 60分/件 | 15分/件 | 75%削減 |
| 月次報告作成時間 | 4時間/月 | 30分/月 | 87%削減 |

## 3. システム化の目的

### 3.1 ビジネス目標

#### 目標1: 業務効率化
- 案件情報の一元管理による検索時間の短縮
- 進捗の可視化による意思決定の迅速化
- 定型業務の自動化

#### 目標2: チーム連携の強化
- エンジニアへの引継ぎプロセスの標準化
- 技術要件の明確化と共有
- コミュニケーションコストの削減

#### 目標3: ナレッジの蓄積
- 過去案件の参照可能化
- 成功事例・失敗事例の共有
- ベストプラクティスの形成

#### 目標4: 顧客満足度の向上
- 迅速な対応による顧客体験の改善
- 提案品質の向上
- 継続的なフォローアップの実現

## 4. システム概要

### 4.1 システムの位置づけ

```
┌─────────────────────────────────────┐
│         DX Consulting CRM           │
│      （案件管理システム）            │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────┐  ┌─────────┐          │
│  │顧客管理  │  │案件管理  │          │
│  └─────────┘  └─────────┘          │
│                                     │
│  ┌─────────┐  ┌─────────┐          │
│  │引継ぎ    │  │進捗記録  │          │
│  │管理      │  │管理      │          │
│  └─────────┘  └─────────┘          │
└─────────────────────────────────────┘
```

### 4.2 システム化の範囲

#### 対象業務
✅ 顧客情報管理  
✅ 案件管理  
✅ 引継ぎ管理  
✅ 進捗記録管理  
✅ ダッシュボード（統計情報表示）  

#### 対象外業務
❌ 請求書発行  
❌ 契約書管理  
❌ 経費精算  
❌ スケジュール管理（カレンダー連携）  
❌ ファイル共有  

※対象外業務は今後の拡張計画に含む

### 4.3 システム利用イメージ

```
【1日の業務フロー】

9:00  ダッシュボードで今日のタスク確認
      → 引継ぎ待ち案件をチェック

10:00 顧客との打ち合わせ
      → 終了後すぐに進捗記録を入力

12:00 エンジニアへの技術相談が必要
      → 引継ぎ記録を作成

14:00 新規顧客からの相談
      → 顧客登録、案件作成、相談内容を記録

16:00 提案書作成
      → 案件の提案内容フィールドに記入

17:00 見積提示
      → 見積金額を入力、ステータスを更新

18:00 1日の振り返り
      → ダッシュボードで全体状況を確認
```

### 5.2 詳細機能仕様

#### F001: 顧客登録

**目的**: 新規顧客情報をシステムに登録する

**入力項目**:
- 会社名（任意、200字以内）
- 担当者名（任意、100文字以内）
- メールアドレス（任意、Email形式、254文字以内）
- 電話番号（任意、20文字以内）
- 業種（任意、100文字以内）
- 企業規模（任意、50文字以内）

**処理内容**:
1. 入力値のバリデーション
2. データベースへの保存
3. 登録日時の自動記録

**出力**:
- 登録完了メッセージ
- 顧客一覧画面へ遷移

**エラー処理**:
- 必須項目未入力: エラーメッセージ表示
- Email形式不正: エラーメッセージ表示

#### F011: 引継ぎ記録登録

**目的**: エンジニアへの技術相談・引継ぎ内容を記録する

**入力項目**:

#### F021: ダッシュボード統計情報表示

**目的**: 案件の全体状況を可視化する

**表示項目**:

1. **統計カード（4枚）**
   - 今月の新規案件数
   - 進行中の案件数
   - 見積総額
   - 引継ぎ待ち件数

2. **ステータス別案件数**
   - 全ステータスの案件数を表形式で表示

3. **最近の案件（10件）**
   - 案件名、顧客、ステータス、見積金額、作成日

4. **エンジニア引継ぎ待ち（5件）**
   - 承認待ちのバトンタッチを表示
   - 案件名、エンジニア名、引継ぎ日、予算

5. **最近の活動記録（10件）**
   - 案件名、活動種別、記録日時、記録者

**計算ロジック**:
```python
# 今月の新規案件数
this_month = timezone.now().replace(day=1)
new_projects = Project.objects.filter(created_at__gte=this_month).count()

# 進行中の案件数
active_projects = Project.objects.filter(
    status__in=['hearing', 'proposal', 'quotation', 
                'negotiation', 'handover', 'in_progress']
).count()

# 見積総額
total_estimated = Project.objects.filter(
    status__in=['quotation', 'negotiation', 'handover', 'in_progress']
).aggregate(total=Sum('estimated_amount'))['total']
```

- INDEX (company_name)
- INDEX (created_at)

#### Project（案件）テーブル

| 列名 | データ型 | 制約 | 説明 |
|------|----------|------|------|
| id | INTEGER | PK, AUTO | 案件ID |
| client_id | INTEGER | FK, NOT NULL | 顧客ID |
| title | VARCHAR(200) | NOT NULL | 案件名 |
| status | VARCHAR(20) | NOT NULL | ステータス |
| consultation_content | TEXT | NULL | ご相談内容 |
| proposal_content | TEXT | NULL | 提案内容 |
| estimated_amount | DECIMAL(10,0) | NULL | 見積金額 |
| start_date | DATE | NULL | 開始予定日 |
| end_date | DATE | NULL | 完了予定日 |
| created_at | DATETIME | NOT NULL | 作成日時 |
| updated_at | DATETIME | NOT NULL | 更新日時 |

**ステータス値**:
- `inquiry`: 初回相談
- `hearing`: ヒアリング中
- `proposal`: 提案作成中
- `quotation`: 見積提示
- `negotiation`: 商談中
- `handover`: エンジニア引継ぎ
- `in_progress`: 実施中
- `completed`: 完了
- `on_hold`: 保留
- `lost`: 失注

**インデックス**:
- PRIMARY KEY (id)
- FOREIGN KEY (client_id) REFERENCES Client(id)
- INDEX (status)
- INDEX (created_at)
